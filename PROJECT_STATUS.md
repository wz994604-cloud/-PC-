## 已知问题

1. 预测历史使用 `/tmp/28live.sqlite` 时属于临时数据，实例重启、冷启动到其他实例或重新部署后可能清空。
2. Serverless 本地 SQLite 无法保证跨实例共享，正式环境需要远程持久化数据库。
3. 每分钟 Cron 已移除；当前依赖接口访问触发累计。长时间无人访问会延迟预测生成和开奖核对。
4. 当前代码尚未建立严格的“开奖前生成截止时间”持久化保障，不能把访问补偿机制等同于可靠高频调度。
5. Preview 在线三个接口、页面一致性、线上幂等与连续 3 期观察未完成。
6. 当前仓库没有足够真实历史数据，因此不能得出 v0.2 优于 v0.1 的结论。
7. 旧状态文档包含不同部署实例与阶段的记录，恢复开发时应以本文件为起点并重新验证外部状态。

## 当前阻塞项

主要阻塞项有两个：

1. **预测历史持久化**：需要接入远程数据库，并迁移开奖记录、预测快照、生成时间、核对状态及模型版本；必须使用唯一约束和事务保持幂等与不可覆盖。
2. **Preview 发布验收**：需要在可授权访问的 Preview 上完成三个接口、页面、重复请求幂等及连续 3 个真实开奖周期观察。

在这两项完成前，不具备部署 Production 的条件。
